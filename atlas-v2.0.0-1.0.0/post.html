Deploying a Website using GitHub Actions CI/CD

March 6, 2026

//

4 min read

In this project, I created a CI/CD pipeline using GitHub Actions to automatically deploy my portfolio website. This ensures that every time I update the code in GitHub, the website is automatically rebuilt and deployed.

Step 1: Creating the Workflow

GitHub Actions workflows are stored inside the .github/workflows folder. I created a workflow file that installs dependencies, builds the project, and deploys the website.

Step 2: Automating Deployment

Whenever new code is pushed to the repository, GitHub Actions automatically triggers the workflow. The pipeline installs dependencies, builds the website, and deploys the updated version to the hosting platform.
